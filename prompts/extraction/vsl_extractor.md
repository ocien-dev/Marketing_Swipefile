# VSL Extractor

Use `prompts/extraction/base_insight_extraction.md` as the output contract.

Focus on extracting VSL intelligence:

- opening lead;
- tension;
- big idea;
- promise;
- unique mechanism;
- problem agitation;
- belief shifts;
- proof sequence;
- offer transition;
- stack;
- guarantee;
- CTA;
- retention and pacing;
- objections inside the VSL.

Prioritize insights that help create:

- complete VSL scripts;
- 3 lead variations;
- mechanism maps;
- proof maps;
- VSL critique checklists.

Special rules:

- If a segment describes sequence, classify as `framework` or `playbook_step`.
- If it gives a reusable VSL block, classify as `template`.
- If it contains actual script language, classify as `partial_copy` or `complete_copy`.
